# Ops — Workflow and deployment

Operational docs for editing and deploying the Material Memory Studio site. Use these with `.cursor/rules/` for consistent, safe updates.

| File | Use |
|------|-----|
| **AGENT_WORKFLOW.md** | How to work on the repo: read docs, inspect HTML, minimal edits, EN/KR and tracking checks, summarize. Common tasks and "do not break" list. |
| **COPY_SOURCE_OF_TRUTH.md** | Canonical messaging and terminology. Reference when writing or editing copy. |
| **DEPLOY_CHECKLIST.md** | Before/during/after editing; pre-deploy browser check; Vercel deploy steps; post-deploy log. |
| **UPDATE_REQUEST_TEMPLATE.md** | Template for requesting updates (goal, page, section, copy, tracking, EN/KR). Copy and fill in when asking for changes. |

Workflow: read `.cursor/rules/` and this README → use UPDATE_REQUEST_TEMPLATE for requests → apply edits → run DEPLOY_CHECKLIST before deploy.
